# Improve IDE Plugin CI with dedicated Gradle jobs

## Problem

Currently, IDE Plugin validation runs as a single bash script job that combines building and testing:
- `.github/workflows/merge.yml` has "IDE Plugin Validation" job
- Uses bash commands to run Gradle tasks
- Slower than using dedicated Gradle GitHub Actions
- No separation between build and test phases
- Less visibility into what's failing (build vs tests)

## Proposed Solution

Replace the monolithic "IDE Plugin Validation" job with two separate, optimized jobs:

### 1. IDE Plugin Build
- Uses `gradle/actions/setup-gradle@v4`
- Caches Gradle dependencies and build outputs
- Runs: `./gradlew :ide-plugin:build -x test`
- Fast feedback on compilation errors
- Uploads build artifacts for test job

### 2. IDE Plugin Unit Tests
- Depends on build job
- Downloads build artifacts
- Uses `gradle/actions/setup-gradle@v4`
- Runs: `./gradlew :ide-plugin:test`
- Uploads test reports on failure

### Benefits

✅ **Faster execution** - Gradle actions have built-in caching (dependencies, build cache, configuration cache)
✅ **Better visibility** - Separate jobs show if build or tests failed
✅ **Parallel execution** - Build can run while other jobs are running
✅ **Reusable artifacts** - Other jobs can depend on build output
✅ **Standard approach** - Matches how other Gradle projects (JUnit Runner, Accessibility Service) are tested

## Implementation Example

```yaml
ide-plugin-build:
  name: Build IDE Plugin
  runs-on: ubuntu-latest
  needs: [detect-changes]
  if: needs.detect-changes.outputs.ide-plugin-changed == 'true'

  steps:
    - name: Checkout code
      uses: actions/checkout@v4

    - name: Set up JDK 21
      uses: actions/setup-java@v4
      with:
        java-version: '21'
        distribution: 'temurin'

    - name: Setup Gradle
      uses: gradle/actions/setup-gradle@v4
      with:
        cache-read-only: false

    - name: Build IDE Plugin
      working-directory: android/ide-plugin
      run: ./gradlew build -x test

    - name: Upload build artifacts
      uses: actions/upload-artifact@v4
      with:
        name: ide-plugin-build
        path: android/ide-plugin/build/libs/
        retention-days: 1

ide-plugin-test:
  name: Run IDE Plugin Unit Tests
  runs-on: ubuntu-latest
  needs: [ide-plugin-build]

  steps:
    - name: Checkout code
      uses: actions/checkout@v4

    - name: Set up JDK 21
      uses: actions/setup-java@v4
      with:
        java-version: '21'
        distribution: 'temurin'

    - name: Setup Gradle
      uses: gradle/actions/setup-gradle@v4
      with:
        cache-read-only: false

    - name: Download build artifacts
      uses: actions/download-artifact@v4
      with:
        name: ide-plugin-build
        path: android/ide-plugin/build/libs/

    - name: Run tests
      working-directory: android/ide-plugin
      run: ./gradlew test

    - name: Upload test reports
      if: failure()
      uses: actions/upload-artifact@v4
      with:
        name: ide-plugin-test-reports
        path: android/ide-plugin/build/reports/tests/
        retention-days: 7
```

## Change Detection

The jobs should only run when IDE plugin code changes:

```yaml
- '**.kt'
- 'android/ide-plugin/**'
- 'gradle/**'
- 'build.gradle.kts'
- 'settings.gradle.kts'
```

## Comparison: Before vs After

### Before (Current)
```yaml
- name: IDE Plugin Validation
  run: |
    cd android/ide-plugin
    ./gradlew build test
```
- Single job
- No caching optimization
- No artifact sharing
- Slower execution

### After (Proposed)
```yaml
ide-plugin-build:
  uses: gradle/actions/setup-gradle@v4
  run: ./gradlew build -x test

ide-plugin-test:
  needs: [ide-plugin-build]
  uses: gradle/actions/setup-gradle@v4
  run: ./gradlew test
```
- Separate jobs with clear responsibilities
- Gradle action caching (dependencies, build cache, configuration cache)
- Build artifacts reused
- Faster parallel execution

## Performance Expectations

Current "IDE Plugin Validation" job: ~3-5 minutes

Expected with optimizations:
- **Build job**: ~2-3 minutes (first run), ~1-2 minutes (cached)
- **Test job**: ~1-2 minutes
- **Total**: Similar or faster due to caching

## Implementation Notes

### Gradle Caching Configuration

Enable configuration cache for even faster builds:

```yaml
- name: Build IDE Plugin
  run: ./gradlew build -x test --configuration-cache
```

### Artifact Optimization

Only upload essential artifacts to speed up job:
- Build: Upload only `build/libs/*.jar`
- Tests: Upload only test reports on failure

### Dependency Chain

```
detect-changes
    ↓
ide-plugin-build
    ↓
ide-plugin-test
```
