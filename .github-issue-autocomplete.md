# Add autocomplete for test plan YAML fields and tool names

## Problem

When authoring test plans in the IDE, users must manually type field names and tool names. This leads to:
- ❌ Typos that trigger validation errors
- ❌ Slow authoring workflow (must reference documentation)
- ❌ Poor discoverability of available tools
- ❌ No hints about required vs optional fields

## Proposed Solution

Add IDE autocomplete (code completion) for test plan YAML files to suggest:
1. **Tool names** - When typing `tool:` in a step, suggest all 59 valid AutoMobile tools
2. **Root-level fields** - Suggest `name`, `steps`, `metadata`, etc.
3. **Step fields** - Inside steps, suggest `tool`, `selector`, `inputs`, `description`, etc.
4. **Metadata fields** - Inside metadata, suggest `version`, `description`, `tags`, etc.

## Implementation

Create `TestPlanCompletionContributor` using IntelliJ's completion API:

```kotlin
package com.automobile.ide.yaml

import com.intellij.codeInsight.completion.*
import com.intellij.codeInsight.lookup.LookupElementBuilder
import com.intellij.patterns.PlatformPatterns
import com.intellij.util.ProcessingContext
import org.jetbrains.yaml.psi.YAMLKeyValue

class TestPlanCompletionContributor : CompletionContributor() {
    init {
        // Complete tool names
        extend(
            CompletionType.BASIC,
            PlatformPatterns.psiElement().withParent(YAMLKeyValue::class.java),
            ToolNameCompletionProvider()
        )

        // Complete field names
        extend(
            CompletionType.BASIC,
            PlatformPatterns.psiElement(),
            FieldNameCompletionProvider()
        )
    }
}

class ToolNameCompletionProvider : CompletionProvider<CompletionParameters>() {
    override fun addCompletions(
        parameters: CompletionParameters,
        context: ProcessingContext,
        result: CompletionResultSet
    ) {
        // Only suggest in test plan files
        if (!TestPlanDetector.isTestPlanFile(parameters.originalFile.virtualFile)) {
            return
        }

        // Check if we're completing a tool value
        val element = parameters.position.parent
        if (element is YAMLKeyValue && element.keyText == "tool") {
            // Add all 59 valid tools
            VALID_TOOLS.forEach { tool ->
                result.addElement(
                    LookupElementBuilder.create(tool)
                        .withIcon(AllIcons.Nodes.Method)
                        .withTypeText("AutoMobile tool", true)
                )
            }
        }
    }
}

class FieldNameCompletionProvider : CompletionProvider<CompletionParameters>() {
    override fun addCompletions(
        parameters: CompletionParameters,
        context: ProcessingContext,
        result: CompletionResultSet
    ) {
        // Suggest field names based on context:
        // - Root level: name, steps, metadata
        // - Inside steps: tool, selector, inputs, description
        // - Inside metadata: version, description, tags
    }
}
```

Register in `plugin.xml`:
```xml
<completion.contributor
    language="yaml"
    implementationClass="com.automobile.ide.yaml.TestPlanCompletionContributor"/>
```

## Benefits

- ✅ **Faster authoring** - No need to reference docs for tool names
- ✅ **Fewer typos** - Autocomplete prevents misspellings
- ✅ **Better discoverability** - See all available tools without leaving IDE
- ✅ **Consistent with IDE patterns** - Uses familiar Ctrl+Space completion
- ✅ **Smart filtering** - Only shows relevant suggestions based on context

## Example User Experience

### Before (Manual typing):
```yaml
steps:
  - tool: tapO|  # User types, gets validation error for "tapO"
```

### After (With autocomplete):
```yaml
steps:
  - tool: tap|  # Ctrl+Space shows: tapOn, swipeOn, pinchOn, dragAndDrop
```

## Acceptance Criteria

- [ ] Autocomplete suggests all 59 valid tool names after `tool:`
- [ ] Autocomplete suggests root-level fields (`name`, `steps`, `metadata`)
- [ ] Autocomplete suggests step fields (`tool`, `selector`, `inputs`, `description`)
- [ ] Autocomplete only activates in test plan YAML files (not all YAML)
- [ ] Autocomplete respects existing text (filters suggestions)
- [ ] Icons differentiate tool names from field names

## Notes

This builds on the existing validation work from #387. The tool registry (`VALID_TOOLS`) and file detection (`TestPlanDetector`) are already in place, so we just need to expose them via IntelliJ's completion API.

Future enhancements could include:
- Smart suggestions based on schema (required vs optional)
- Selector autocomplete based on current screen
- Input parameter suggestions based on tool type
